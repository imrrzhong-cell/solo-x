# MyAIHOT 部署 — 手动待办

> 代码已推送到 main，Vercel 自动部署中。以下 3 步需要你手动完成。
> 完成后访问 `https://你的域名/webapps/aihot/featured` 即可看到数据。

---

## 第 1 步：创建 Neon PostgreSQL 数据库

1. 打开 https://neon.tech 并登录/注册
2. 创建新项目，区域选 **Singapore (ap-southeast-1)**
3. 进入项目后，复制连接串（选择 **Connection string** → **Node.js** 格式）：
   ```
   postgresql://user:pass@ep-xxx.neon.tech/myaihot?sslmode=require
   ```
4. 打开 Neon SQL Editor，执行：
   ```sql
   CREATE EXTENSION IF NOT EXISTS vector;
   ```
5. 记下连接串，后面两步都要用

---

## 第 2 步：部署 Python Worker 到 Fly.io

Worker 代码位置：`/Users/rustyryan/Desktop/files/myaihot-worker/`

```bash
cd /Users/rustyryan/Desktop/files/myaihot-worker

# 1. 登录 Fly.io（如未安装：brew install flyctl）
flyctl auth login

# 2. 创建应用
flyctl apps create myaihot-worker

# 3. 创建持久卷（缓存模型用）
flyctl volumes create model_cache --region sin --size 1

# 4. 设置环境变量（替换下面的占位符）
flyctl secrets set \
  DATABASE_URL="postgresql+asyncpg://user:pass@ep-xxx.neon.tech/myaihot?sslmode=require" \
  ANTHROPIC_API_KEY="你的 Anthropic API Key" \
  WORKER_API_KEY="生成一个随机密钥，比如 openssl rand -hex 32"

# 5. 部署
flyctl deploy

# 6. 验证健康检查
curl https://myaihot-worker.fly.dev/health

# 7. 运行数据库迁移
flyctl ssh console --command "alembic upgrade head"

# 8. 导入初始信源
flyctl ssh console --command "python scripts/seed_sources.py"
```

> **注意**：DATABASE_URL 这里用 asyncpg 格式（`postgresql+asyncpg://`），和第 1 步拿到的略有不同，需要手动加 `+asyncpg`。

---

## 第 3 步：配置 Vercel 环境变量

1. 打开 Vercel Dashboard → 选择 SOLO.X 项目 → **Settings** → **Environment Variables**
2. 添加以下变量（Production + Preview + Development 全选）：

| 变量名 | 值 | 说明 |
|--------|-----|------|
| `DATABASE_URL` | `postgresql://user:pass@ep-xxx.neon.tech/myaihot?sslmode=require` | 第 1 步的连接串（Node.js 格式，**不带** asyncpg） |
| `WORKER_API_URL` | `https://myaihot-worker.fly.dev` | 第 2 步部署的 Worker 地址 |
| `WORKER_API_KEY` | 第 2 步生成的随机密钥 | 需与 Worker 端一致 |
| `CRON_SECRET` | `openssl rand -hex 32` 生成 | 保护 cron 端点 |

3. 可选：`AIHOT_ADMIN_PASSWORD`（保护写操作如收藏/反馈）
4. 保存后，Vercel 会自动重新部署

## 部署后验证

全部完成后，按顺序测试：

```bash
# 1. 手动触发数据流水线
curl -H "Authorization: Bearer 你的CRON_SECRET" \
  https://你的域名/api/aihot/cron/pipeline

# 2. 检查 Worker 日志
flyctl logs

# 3. 查看精选页
open https://你的域名/webapps/aihot/featured

# 4. 手动触发日报
curl -H "Authorization: Bearer 你的CRON_SECRET" \
  https://你的域名/api/aihot/cron/daily-report
```
